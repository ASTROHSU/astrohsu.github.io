Amazon 的中立承諾

過去 5 年，每篇區塊勢文章都會附上一段朗讀音檔。那是我在寫完文章、編輯改完定稿之後，深夜再花半小時錄製的內容。累積下來，已經有四百多篇了。這週開始，我想嘗試一個新做法：用 Google 的 AI 工具 NotebookLM，把每篇文章轉成繁體中文的「對話音檔」，暫時取代我的真人朗讀。對內容創作者來說，這是一種榮耀 —— 內容不只是被讀，還能被「討論」。一個月後，我會再來問大家感受，看看要不要繼續。

另外提醒一下，很多人可能不知道可以用 RSS Feed 把區塊勢連到 Apple Podcasts 或 Spotify。以後開車、運動的時候，就能直接用聽的追文章，比較方便。這週的主題也與 Google 有關。

上週 Google 宣布推出自家研發的獨立區塊鏈 Google Cloud Universal Ledger（簡稱 GCUL）。有趣的是，Google 的 Web3 主管就像是有在讀區塊勢一樣，對於時事的見解相同。他說：「Tether 不會用 Circle 推出的區塊鏈，而 Adyen 也可能不會用 Stripe 的區塊鏈。但任何金融機構都可以建立在 Google 鏈上」。

Google 稱全球最大的商品期貨交易所 —— 芝加哥商品交易所（CME）—— 就選擇將代幣化、支付實驗跑在 Google 鏈上。Google 鏈主打三大特色：高效能、支援 Python 以及可信中立（credibly neutral）。相信沒人會懷疑 Google 達成前兩項特色的技術實力，這篇文章只討論最後一點，也就是 Google 如何做到可信中立，以及它對區塊鏈有多重要？但我要先從全球最大電商平台 Amazon 的故事說起。

Amazon 的中立承諾

Amazon 成立於 1995 年，公司原本取名為 Cadabra，因為律師一度聽成 cadaver（屍體），才改名為 Amazon。創辦人 Jeff Bezos 選擇這個名字有兩個原因：「A」在字母當中排序最前面，最容易被看見；此外，亞馬遜河是世界上流域最廣的河流，Bezos 希望 Amazon 也能成為世界上最大的網路書店。

創立初期，Amazon 是書店和出版社的「好朋友」，就是幫忙把書放到網路上賣。網站的運作模式也很輕量，Amazon 收到網路訂單後才向出版商採購，再寄給消費者。這套模式很快就吸引其他類型的賣家，拜託 Amazon 幫忙開拓線上通路。可以說 Amazon 早期的成功秘訣就是專心協助賣家做生意，更把自己定位成中立的電商平台，不和賣家直接競爭。

但隨著生意越做越大，這個「不競爭」承諾在 2009 年開始動搖。那年 Amazon 推出自有品牌 Amazon Basics，產品設計雖然不搶眼，但勝在便宜、實用。最經典的例子就是電池。電池是日常必需品、銷量穩定，Amazon Basics 電池「俗擱大碗」，平台搜尋結果常常排在金頂（Duracell）、勁量（Energizer）等知名品牌之前。憑藉低價、曝光雙重優勢，Amazon Basics 電池 銷售額一路攀升，甚至在 2018 年拿下美國線上電池市場 30% 市占率。

這時平台上的商家才驚覺 Amazon 當年的承諾早已不在。更殘酷的是 Amazon 不只下場競爭，還透過平台的銷售數據精準判斷，決定下一個 Amazon Basics 要複製哪款熱銷商品。

平台變卦不是新鮮事，承諾不變才是新聞。上週 Google 宣布推出區塊鏈 Google Cloud Universal Ledger，號稱是「可信中立」（credibly neutral）的公共基礎建設。Google 承諾不會下場競爭，而是要專心幫助金融機構把錢搬到鏈上。

Google 鏈的意義

要理解 Google 鏈，必須先從 Google Cloud 說起。Google 的 Web3 業務長期隸屬於 Google Cloud 部門，這直接決定了 Google 看待區塊鏈的方式，不是怎麼從零打造鏈上的去中心化應用，而是怎麼做區塊鏈生意，幫 Google Cloud 賺錢。

過去幾年 Google 提供各家區塊鏈的節點託管、區塊鏈 API 服務，並協助交易所、DeFi 開發者以 Google Cloud 運行應用，本質上是「軍火商」。但現在 Google 推出 Google Cloud Universal Ledger（GCUL），等於自己下場。對於這樣的轉折，Google 在官方部落格解釋，現有金融系統零碎、昂貴且緩慢，而未來的金融基礎建設很可能會建立在區塊鏈上，因此 Google 要做的就是這個新基礎建設的供應商。

這是一種彎道超車策略。雲端市場有三大供應商：亞馬遜 AWS、微軟 Azure 和 Google Cloud，Google 市占率落後前兩者。恰好現在各家金融機構都在思考是否應該上鏈。如果要，那該上什麼鏈？以往 Google 會攤開一張他們支援的區塊鏈列表，要金融機構從中選一條。但同樣的問題被問久了，Google 自然也會想：「為什麼我不自己來做一條區塊鏈？」

對金融機構來說，選擇也變得簡單許多。以 CME 為例，他們本來就跟 Google Cloud 簽了 10 年的雲端合約。既然本來就是用 Google 的服務，現在要使用由 Google 推出的鏈測試代幣化也就順理成章。相較之下，若要搬去以太坊、Solana，不僅技術環境和運作模式都和 Web2 雲端天差地遠，要說服主管同意更是難上加難。理解這些背景後，再來看 Google 鏈的定位就會清楚許多。

Google 將 GCUL 定義為一套受監管、許可制的區塊鏈。既然不是開放式的公有鏈，就不必設計代幣經濟模型，也不用考慮「如何激勵礦工」這類難題。因此 Google 很可能不會發行自己的加密貨幣，用戶只要按照原本的雲端計價模式付費即可。而 GCUL 最主打的賣點是「可信中立」，意思是不會像 Circle、Stripe 的區塊鏈一樣獨厚自己人。Google 承諾 GCUL 會是中立的結算平台。只專注在降低成本、提高效率，不會和金融客戶搶生意。

這是把鏈當成雲來賣。金融業是雲端市場利潤最豐厚，且最不輕易換基礎建設的金雞母。對 Google 而言，推出 GCUL 的核心目標是要擴張 Google Cloud 在金融業的市占率，而不是去中心化。Google 批評傳統金融緩慢、效率低落，可能只是因為 Google Cloud 在那裡的市占率不夠高、沒有包袱罷了。雖然 Google 承諾自己不會偏心，但誰能保證它未來不會切入金融領域？

不作惡

Google 沒有銀行牌照，短期內能涉足的金融業務確實有限。但科技與金融結合卻是大勢所趨。

知名創投 a16z 的合夥人 Angela Strange 曾預言「所有公司終將成為金融科技公司」，提供金融服務不僅可以提升公司獲利，還能夠提升客戶黏著度。例如 Shopify 原本只是幫商家在網路上開店並收取月費1，後來發現做生意最重要的就是收錢、借錢，於是推出支付、貸款服務。如今金融業務已占 Shopify 收入將近一半，商家也更加依賴平台。

Uber 則是推出司機專屬的金融帳戶和 Uber Pro 借記卡，能享有現金回饋、加油折扣與維修優惠。這些服務不只增加司機對平台的滿意度，也降低他們跳槽到競爭對手的可能性。近期 Stripe 宣布推出「穩定幣即服務2」更是關鍵。任何公司只要串上 API 就能發行自家穩定幣，進一步降低參與門檻。

假如未來蘋果、臉書、亞馬遜都開始發行穩定幣，Google 是否還能維持「不與客戶競爭」的界線？GCUL 掌握的龐大金融交易數據，會不會成為下一個 Amazon Basics？

以太坊創辦人 Vitalik Buterin 曾提出「可信中立」的定義：不偏袒任何人，而且公開透明、人人可驗證。Google 所強調的「可信中立」則完全不同，它不是一套制度化的規則，而是科技公司的承諾。這更像是 Google 早期喊出的口號「不作惡」（Don’t be evil），要大家相信 Google 會在關鍵時刻做對的事。但公司使命是為股東創造利潤，而不是維護公共利益。

雖然 GCUL 號稱是 L1 區塊鏈，但競爭場域其實是 Stripe、Circle 這些由企業打造的「區域網路3」，而不是以太坊這樣的「網際網路」。這個差異非常關鍵，我們未來使用的金融基礎建設等於是有兩個完全不同的走向，一個是由企業承諾維護的區域網路，另一個則是制度設計下、任何人都能驗證的公共基礎建設。

由私人公司建造的公共建設，或許能蓋出社區的中庭花園，但終究無法成為通往世界的道路與橋梁。

p.s. 音檔裡面有時候會說 GCUL 有時候是 GCLU，但他們都在說同一件事 😂
