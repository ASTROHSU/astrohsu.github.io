以太坊的三套劇本

L2 擴容路線過時了？

向下滑動
PART 1

Vitalik 的震撼彈

最近幣價表現不怎樣，又到了考驗信仰的時刻。但即便你信仰的是技術，還是有可能踢到鐵板。過年前，以太坊創辦人 Vitalik Buterin 拋出震撼彈，稱過去五年以 L2 為中心的擴容劇本已經過時。加上預期今年以太坊主網（L1）的效能會大幅提升，他提議調整發展重點。Vitalik 更直言，L2 必須找出「擴容」以外的獨特價值，因為未來以太坊主網的手續費就已經夠便宜。

但這番言論讓 L2 情何以堪，社群隨即反彈。Arbitrum 創辦人就公開與以太坊劃清界線，稱「Arbitrum 不是以太坊」。反倒是曾積極爭取 L2 稱號的 Polygon 幽默發文：「因應市場風向，從今天起我們將改以側鏈自居」。這不只是社群吐槽，而是反映了一個更深層的問題：五年的技術投資，是否化為泡沫？或者以太坊的發展策略真的走到了轉折點？

要理解 Vitalik 為何忽然態度大轉彎，就得從當初為何會有 L2 說起。那個決定改變了整個區塊鏈的生態，現在卻可能面臨根本性的修正。

PART 2

當初為什麼選 L2

以太坊誕生於 2015 年，從一開始就把系統韌性擺在首位，效率反而是其次。它優先考慮的是抗審查、不可逆與去中心化，交易吞吐量排在後面。就像台灣選擇民主體制，也是把容納多元意見與權力制衡擺在決策效率之前。但現實是，民主不能當飯吃。鏈上手續費居高不下，多數人都用不起。於是，開發者早在 2017 年就開始思考如何兼顧效率與去中心化。Vitalik 提出區塊鏈的「不可能三角」理論，精準點出核心矛盾：安全性、去中心化與擴容，難以同時兼得。

後來，檯面上逐漸浮出三套擴容劇本：提升硬體效能、推進分片技術，或是把使用者搬到第二層網路。第一條路最直覺，但會犧牲去中心化；第二條路理論完整，但技術門檻極高。最後，以太坊選擇第三條路 —— 把相對次要的「效率」外包給 L2 處理，讓主網專注安全性與去中心化。

過去幾年，由第三方打造以太坊 L2 如雨後春筍出現。Arbitrum、Base、Polygon 迅速崛起，各大交易所也陸續支援。當時 Vitalik 的想像是，每條 L2 都是以太坊文化的延伸。雖然各自發展，但最終仍然同屬以太坊區塊鏈。以太坊就像是「數位聯邦」。主網是聯邦政府，負責共識與安全。而 L2 則像州政府，在各自領域開疆闢土。誰能想到五年後，Vitalik 竟然親口宣告這套劇本已經過時。

核心觀點

三套劇本的未來走向

Vitalik 認為目前最可能的是劇本 3，但他更希望看到劇本 2。劇本 1 意味著 L2 從此不再屬於以太坊。

PART 3

新的擴容劇本

Vitalik 提出修改路線圖的原因很明確：「以 L2 為中心的路線圖已經過時了。其中有兩項關鍵原因：第一，L1 的擴容進展比想像的更快。第二，L2 的去中心化進展比預期的更慢。」當初以太坊選擇發展 L2，是妥協於技術上的「不可能三角」。既然一條鏈難以同時兼顧安全、去中心化與效能，那就把不同功能拆開來處理。但這幾年的技術進展 —— 尤其是零知識證明的成熟 —— 讓 Vitalik 意識到，以太坊主網或許已經快要有辦法自行突破「不可能三角」。

最終讓 Vitalik 決定修改路線圖的原因，是他察覺到有些 L2 進展緩慢，不是出於技術瓶頸，而是因為參雜了商業與政治考量。某些 L2 團隊明確表示，他們永遠不會走向去中心化。這不僅是出於技術顧慮，也是顧及客戶的監管需求，要求 L2 保有最終控制權。這對他們的客戶而言或許是正確選擇，但如果選擇這麼做，那麼他們根本不是在做原始意義上的「擴展以太坊」。

字裡行間看得出來，Vitalik 有點不耐煩。以太坊始終把「去中心化」擺在首位，寧可犧牲效率也不妥協。沒想到有些 L2 靠著「高效率、低手續費」成功圈到一批使用者後，就開始各懷鬼胎。既然使用者和應用都在我這裡，保留區塊鏈的控制權更符合商業利益和監管需求，為什麼一定要走向完全去中心化？這直接踩中 Vitalik 的底線。這些 L2 已經本末倒置，根本不是在擴展以太坊。

「既然 L1 的效能可望大幅提升，擴容的任務就不必全押在 L2 身上。一旦主網的 gas fee 夠便宜，使用者大可以忽略 L2 的存在。」

PART 4

開發者的信任危機

在過去幾年，最難解釋的問題是：「為什麼我的 ETH 上面寫著 Base？」至今還沒辦法用一句話講清楚。只能含糊地說，那就是手續費比較便宜的 ETH。心裡還會暗自祈禱，希望對方不要把這些 ETH 轉入中心化交易所，因為八成會轉錯鏈。

這幾年交易所客服每天都在處理轉錯鏈的工單。開發者花大量時間設計跨鏈橋、做帳戶抽象化，就是為了把 L2 的複雜性藏起來。大家心知肚明，L2 嚴重破壞使用體驗，才衍生出這些問題。當初為了壓低 gas fee，大家把它當成必要之惡。

如果未來技術進展真如 Vitalik 所預期那樣順利，眼前許多關於使用體驗的研發工作，或許就不再是當務之急。以後可能只有進階使用者才需要深入理解 L2，多數人只要安心使用以太坊就好。但這對投入巨資開發 L2 的團隊而言，無疑是個沉重打擊。這也難怪 L2 團隊會這麼生氣。雖然這不完全是 Vitalik 的問題，但肯定會傷害信任。有了這次經驗，未來開發者在投入資源前可能會先確認：萬一路線圖再次改寫，自己會不會成為下一個技術孤兒？

三套劇本解析
🌱

劇本 1：L2 專層化

L2 放棄「以太坊第二層」的身份，轉向打造專用應用鏈。尋找擴容以外的獨特價值，如特定領域優化、隱私機制或垂直整合。實際上已在發生。

⚡

劇本 2：L1 吸收

以太坊主網透過零知識證明、分片等技術大幅提升效能，手續費趨近於零。L2 的效率優勢消失，大多數項目遷回主網。基礎設施簡化。

🤝

劇本 3：混合共存

L1 提升到可用水準，L2 則找到差異化角色（DeFi 專層、隱私層等）。兩者共存競爭，市場自然選擇最優方案。最符合開放生態精神。

你認為哪套劇本最可能發生？

展望以太坊未來，你的看法如何？

劇本 1：L2 專層化

你認為 L2 具有足夠的獨立生存力，能夠找到超越「低手續費」的差異化價值。這個看法指向一個更去中心化的生態，每條鏈各自演進，就像真正的多鏈世界。

劇本 2：L1 吸收一切

你對以太坊的技術進展樂觀，相信零知識證明和其他創新足以讓主網突破瓶頸。這意味著未來使用者體驗會大幅改善，但同時也意味著 L2 的生存空間被擠壓。

劇本 3：混合共存

你認為這是最務實的未來。L1 和 L2 各有所長，會逐漸形成一個成熟的生態系統，每種應用尋找最適合的執行環境。這反映了一個成熟市場的自然演進。

L1 與 L2 的關係正在被重新定義。

Vitalik 的轉變不只是技術方向的調整，更反映了整個以太坊社群對未來的新認知。
