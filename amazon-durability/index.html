PART 1 — 十年預言成真

Amazon 把物流網
賣給對手了

5 月 4 日 Amazon 推出 Amazon Supply Chain Services(ASCS),把空運、海運、卡車、最後一哩的全套物流能力打包成服務,賣給寶僑、3M 這些原本不算 Amazon 客戶的大公司。FedEx 和 UPS 的股價當天應聲下跌。

這個動作看起來像是新聞,其實是十年前的劇本被精準演完。2015 年 Ben Thompson 寫過一篇〈The Amazon Tax〉,當時 Amazon 只剛開始試水溫式地租了六架貨機,Thompson 就預測:Amazon 會把這套物流網路賣給其他公司,就像它把自家的伺服器資源賣成 AWS 一樣。

Amazon 做事的公式始終一致。先在自家業務裡蓋出最基礎的能力(primitives),Amazon 自己當第一個、也是最重要的客戶,用內部需求的規模把建置成本攤掉。等到這套能力跑得夠成熟,再把它打包賣給外面的人。賣得越多、規模越大,單位成本越低,護城河就越深。AWS 是這樣、電商履行(FBA)是這樣,現在是物流。

Amazon 三條長期投資的時間線
2006
AWS 對外開放 從支撐 Amazon.com 自身運算需求,變成全世界最大的雲端服務商。
2014
Amazon 開始建貨機隊 從 6 架到 20 架到上百架,逐步擺脫對 UPS、FedEx 的依賴。
2015
收購 Annapurna Labs 為日後自製 Graviton、Trainium 晶片埋下伏筆。
2026
ASCS 對外開放 物流網從 Amazon 自用工具,變成第三方付費使用的服務。
區塊勢製圖 · 資料來源:Stratechery / Bloomberg

物流被打開的那一刻,Amazon 完成了「邊際成本變成資本成本」的轉換。每送出一個包裹,都不再是花錢給 FedEx 或 UPS,而是攤平自家折舊。其他公司付錢來用,只是讓這個資產的回報率變得更高。

PART 2 — 三年前看走眼的人

AWS 看似要被 AI
淘汰的那段日子

2023 年 ChatGPT 引爆 AI 熱潮的時候,半導體研究機構 SemiAnalysis 寫了一篇文章:〈Amazon 的雲端危機:AWS 將如何輸掉運算的未來〉。當時的論點看起來相當有說服力。

SemiAnalysis 點出 AWS 的兩個結構性問題。第一,AWS 堅持用自家的 Nitro 和 EFA 網路架構,而不採用 Nvidia 或 Broadcom 的最佳網路方案。在大型語言模型訓練時代,你需要把上千顆 GPU 串成一個系統,GPU 之間的橫向通訊速度是關鍵,AWS 的網路架構就是不夠用。第二,Nvidia 在缺貨年代擁有絕對話語權,而 AWS 又一直想自製晶片擺脫 Nvidia,Nvidia 自然就把 H100 優先分給其他更聽話的雲端商。Microsoft Azure、Oracle、CoreWeave 那段時間都比 AWS 拿到更多 GPU。

這些觀察當時都對。問題在於,他們沒有預期到 AI 工作負載的本質會變。

AI 算力市場的兩次切換
2023 主場:訓練 需要把上千顆 GPU 串成單一系統,GPU 之間要能用最快的網路即時同步權重。Nvidia 的整套網路方案幾乎是唯一選擇,AWS 的客製架構派不上用場。
2026 主場:推理 模型參數可以塞進單一伺服器,不需要把 GPU 串成大叢集。Agentic 工作負載還需要大量 CPU 協調,正好是 AWS 用 Nitro + Graviton 擅長的拆解式架構。

轉折點是 ChatGPT 之後的兩個演化。先是 o1 等推理模型出現,模型在回答前要先「想」很多 token,推理算力需求暴增。接著是 Agent 出現,人可以同時開好幾個 Agent,每個 Agent 又會多次呼叫推理模型。這不是線性增加,是兩個指數疊乘。整個 AI 算力市場的重心,從訓練快速移到推理。

這個轉移對 AWS 是天大的好消息。推理不需要把上千顆 GPU 串在一起,單台伺服器就能處理。Agent 大量依賴 CPU 來協調,而且要在 CPU 和加速器之間靈活分派工作。這些特性正好對上 AWS 的 Nitro 架構,把運算、儲存、網路、虛擬化拆成獨立模組,然後用便宜的自家 ARM 晶片(Graviton)當 CPU、用自家的 Trainium 當推理加速器。

2015 年買下 Annapurna Labs、2019 年推出第一代 AI 晶片時,Trainium 慢得讓人尷尬。但 Amazon 沒停,一代一代迭代,現在 Trainium 3 已經可以打。這就是長期主義的優勢。重要的不是某一代晶片好不好,而是這條曲線已經走了七年。

PART 3 — Anthropic 為什麼變關鍵

基礎模型的
中立平台價值

AWS 還有一個外人看不到的優勢:Anthropic。

Nvidia 創辦人黃仁勳前陣子接受 Dwarkesh Patel 訪談時自承,沒有早期投資 Anthropic 是個錯誤。他的解釋很坦白:當時 Nvidia 沒有條件砸幾十億美元到一家 AI 實驗室,而 Google 和 AWS 都有。前期投資讓 Anthropic 反過來把運算花費投回 AWS 和 GCP,這是一個正向循環,而 Nvidia 沒有參與。

Amazon 之所以投得起 Anthropic,正是因為 AWS 多年累積的現金。長期投資的另一個複利效應:基礎建設的回報能轉投資成下一波關鍵卡位。

AWS CEO Matt Garman:大多數客戶根本不在意自己用的是哪一顆加速器晶片。你跟 OpenAI 對話時不會去碰到 GPU,你跟 Claude 對話時也不會碰到 Trainium 或 TPU,你只會碰到介面。能訓練前緣模型的團隊本來就只有那幾家。 — Stratechery 訪談

這帶出一個有趣的觀察:在 AI 競賽裡,模型本身是中立資產的一方,反而比擁有自家模型的一方更安全。

Microsoft 過去靠 OpenAI 獨家綁定 Azure,但這個獨家現在反而是負擔。Microsoft 自己是軟體公司,要對 Copilot、Office 365 投入大量算力,結果今年 Q1 Azure 雲端營收就因為內部 AI 工作負載排擠掉客戶用量,沒達到預期。Google 也類似,它的核心是搜尋,搜尋面對 AI Chatbot 的長期挑戰存在,內部 AI 投資的優先順序高過租給客戶。

Amazon 不一樣。Amazon 的核心業務是賣實體商品和蓋資料中心,這兩件事跟「我自己要把 AI 算力都拿去用」沒有直接衝突。Amazon 願意把絕大多數晶片留給雲端客戶,這在受限算力的時代是個珍稀的賣點。Anthropic 又是當前唯一在三大雲端商都有完整部署的前緣模型,企業客戶要選 AI 雲端時,Amazon 的組合最沒有政治顧慮。

PART 4 — 衛星、無人機、實體護城河

Leo 衛星的
真正用途是無人機

Amazon 還有一個看似奇怪的長期投資:Leo 衛星網路。表面上看是要跟 SpaceX 的 Starlink 競爭,而 Starlink 已經在天上飛了很多年。為什麼 Amazon 還要花這麼多錢蓋一個重複的東西?

CEO Andy Jassy 在最近一次法說會上的描述,跟他談 AWS 的口吻一模一樣:前期需要砸大量資本和現金、蓋資產,然後在中長期享受很高的自由現金流和投資報酬率。這就是 Amazon 的固定公式。把邊際成本轉換成資本成本,再用規模把這個資本攤掉。

但 Leo 真正的賣點不只是賣衛星網路。Amazon 在 2013 年就公開談過無人機送貨。十三年過去,雖然進度比想像中慢,但要看到一個圖像越來越清楚:當無人機真的能取代人力配送時,送貨成本就只剩下無人機這個資產的折舊,不再有人事費、油料費、最後一哩的時間成本。這跟 AWS 把伺服器折舊變成主要成本是同一個邏輯。

那麼,當無人機飛在天上的時候,要靠什麼跟它通訊?衛星網路。Amazon 不想被黃仁勳卡晶片,當然也不想被馬斯克卡無人機通訊。Leo 不是為了搶 Starlink 的客戶,是為了確保 Amazon 自己的下一個基礎建設不會被別人卡住。蓋好之後,還是可以順便賣給 Apple 或其他公司,延續同一套商業模式。

PART 5 — 真正的洞察

越紮根實體世界的公司
越不怕 AI

把 Amazon 的故事拉開來看,可以得到一個對整個科技業的判斷標準:一家公司面對 AI 衝擊的長期脆弱度,跟它的業務有多紮根於實體世界成反比。

📦
Amazon、Apple
紮根實體 · 最安全
不需要擁有最強模型,只要能用得到就好。核心業務是賣商品、賣硬體、蓋資料中心,AI 是工具不是命脈。
💼
Microsoft
中等暴露
沒有自家前緣模型,但對企業客戶的通路掌握度高。投資資料中心,但 OpenAI 依賴是潛在風險。
🌐
Google、Meta
數位平台 · 最暴露
必須投資自家模型到跟 Amazon 同等規模。注意力一個點擊就被換走,AI 對核心業務是存亡威脅。

Apple 一直被批評 AI 動作太慢,但從這個角度看,Apple 跟 Amazon 都不需要趕。它們的核心是賣硬體、賣商品,有訪問前緣模型的能力就夠了。模型自己會競爭、會降價,Apple 可以從容選擇。Microsoft 在通路上夠強,但內部 AI 投資的優先順序壓過雲端,代表它的核心軟體業務感受到了真正的壓力。Google 和 Meta 是純粹的數位聚合者,使用者注意力是它們唯一的資產,AI 對它們來說就是存亡。所以這兩家公司的資本支出規模直追 Amazon,而且都堅持自研模型,沒得選。

這也回到 Amazon 為什麼能玩十年週期的長賭局。當別人面對的是「兩三年內必須轉型不然就死」的時間壓力,Amazon 面對的是「十年內慢慢蓋好,蓋好了就能用很久」的悠閒節奏。當下看不出來哪邊厲害,十年回頭看才看得清楚。

越是紮根實體世界的公司,越不怕 AI 革命。

Amazon 用十年慢慢蓋,黃仁勳和馬斯克都會被它繞過。
